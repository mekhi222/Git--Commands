# clone
- creates a local copy of a remote repository on your machine.



# push
- git push command allows you to send (or push) the commits from your local branch in your local Git repository to the remote repository.

# checkout 
- Switches to a different branch or restores files from a commit.

# B
- b Used with checkout (or switch) to create a new branch and switch to it immediately.

# Status
- Shows the current state of your working directory and staging area. 

# P
u – Sets an upstream branch so future pushes/pulls can be done without specifying the remote/branch.

# pull 
pull – Fetches changes from a remote repository and merges them into your current branch.

# add – 
- Stages changes to be included in the next commit.

# commit –
- Saves staged changes as a snapshot in the repository history.

# -m – 
-Lets you include a commit message directly in the command line.

# a – 
-Automatically stages all modified tracked files before committing.

# am – 
-Combines -a and -m to stage changes and add a message in one 
command.
# branch – 
-Lists, creates, or deletes branches.

# -a 
-(with branch) – Shows all branches, including remote ones.

# d – 
-Deletes a branch safely (only if it has been merged).

# merge
- – Combines changes from one branch into another.
